Project 1: The Elevator Interface


This GitPage was made to cover key project requirements


CS 3366 Human Computer Interaction

Cameron T Emfinger

TTU Fall 2022

Grade C Requirements


Issues with current interfaces

No indication of the current floor from the outside or inside of the elevator.

The "Alarm" and "Stop" buttons are both too close to other buttons and could be pressed accidentally.

No braille on the outside interface for the visually impaired.

Pressing a floor that you are already on will not open the door.


What is good or bad here?

Good:

Simple Interface

Nice Tactile and Visual feedback for buttons

Open door button actually works


Bad:

It "Feels" Old

Could accidentally push the Emergency buttons

No Indication of what floor the elevator is on from the interfaces

No braille on the outside interface

Grade B Requirements


Some commonly used things:

    A button on the outside of the elevator to indicate if you want to call the elevator to go up or down.

    A button to close and open the doors.

    Multiple buttons to declare what floor you want to go to.

    A railing to hold onto in the elavator.

    A visual indicator of what floor the elevator is currently on from both the outside and inside.


Some more rarely used things:

    A button to stop the elevator.

    A button to sound an alarm for help incase of emergencies.

    A button or phone to speak with a 24-hour monitoring service


Does the interface make doing these common things easier?


Yes the existing iterface covers the common interactions quite well.


A common sequence of actions

  1. User presses button to call elevator
  2. User waits for elevator
  3. Elevator door opens
  4. User walks into elevator
  5. User presses button to declare what floor they want to go to
  6. User waits for elevator
  7. Elevator door opens
  8. User arrives to desired floor

How does the elevator support the user to figure out how it works?

The current interface is simple in design. From the outside interface you have an arrow pointing up which infers that when you press that button the elevator will be going up when you get in.


From the inside controls (like most elevators) are just implied. If you press 2 you are expected to go to floor two.


User Feedback

    Visual feedback when buttons light up after pressed

    Tactile feedback when pressing button


Common possible mistakes.

    Accidentally pressing the elevator stop or alarm buttons

    No conformation for alarm or stop


Suggested Improvements:

Changing interface to touchscreen

Putting "Destination" by the buttons

Having a current floor indicator

Including the Phone system into the interface

Tapping Emrgency or Phone will send you to another screen
